00: group data and funcitons together:
	OBJECT - concrete variable created from a class.

	C++ object is close to a struct variable, but stronger:
	-it holds data
	-it can also have functions attached to it
	-it can automatically initialize and clean itself

	CONSTRUCTOR - function that runs automatically when obj is created:
	it is an object's starting setup function.
	Constructor with reference memeber must use init list.

	METHOD - function that belongs to a class and is called on an obj.
	meaning is tied to an object.

01: visualizing memory: where objects live and how to access them.

	STACK - HEAP - REERENCES -POINTERS

	reference - must be set at construction, always exist
	pointer - can be null and set later

STACK → automatic lifetime
HEAP  → manual lifetime


	* brace uniform initialization unifies older init syntaxes:
		/ copy std::string s = "hi";
		/ direct std::string s("hi");
		/ aggregate int arr[3] = {1, 2, 3};
		/ brace std::string s{"hi"}; int arr{1, 2, 3};
		- it catches bug such as narrowing conversions int x{2.3}; decimal part doesn't get lost silently.
		- it works well with containers: std::vector<int> v{1,2,3};
