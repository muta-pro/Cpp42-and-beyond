ex03: introduces multiple inheritance- > virtual inharitance
	classic diamond inheritance shape.
 - multiple inheritance creates ambiguity
 	concept: class can inherit traits from more derived classes-> trait composition.
 	- it has 2 Names: it's own and inherited to show that an obj has his own
 	identity but also a base one.
 -> derived class may keep its own state while also managing inherited state explicitly

 conflict resolution between parent classes: in multiple inherit. u don't take the whole block,
 but the parent's version is chosen -> attack() from ScavTrap class.
 so : it is a ClapTrap class with with data and behaviout from child classes available while
 having it's own identity.
  -> WhoAmI() clearly tests the multiple identity tracking.

  virtual: solves duplicate base subobject ambiguity -> one shared ClapTrap base
  explicit qualification solves same-function-name ambiguity

  Once ClapTrap is virtual base -> hybrid class must contruct it:
  	 because two subclasses inherent the base class as virtual public class, then
  	 it means that base class is virtual base, for that reason hybrid class is
  	 responsible for constructing the shared parent class subobject.

 Hybridclass() : Parentclass(), Subclass(), _att() {}

 	so, order of the initializer  does not define the construction order.
 	C++ construction order:

 	1 - virtual bases
 	2 - direct bases the order they are declared in the classes
 	3 - memeber in order they are declared
 	4 - constructor body
